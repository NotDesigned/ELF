### 复现设计

当前代码在每个 sampling family 内保存 RNG 状态，并在 oracle 与 shuffled 两种干预前恢复，
因此二者使用相同 token-noise stream。自由联合生成仍只作为质量参照，不作为与 oracle 的
严格同噪声因果对照。

1. 兼容性 smoke：seed 42、32 steps、16 samples。
2. 正式评测：seed 42/123/456 × 32/64 steps，每格256 samples。
3. 32-step family 保持历史参数：SDE、CFG 1、SC-CFG 3、logit-normal、gamma 1.5。
4. 64-step family 保持历史参数：SDE、CFG 1、SC-CFG 3、logit-normal、gamma 1.0。
5. 每格记录 `g_ppl`、`oracle_plan_ppl`、`shuffled_plan_ppl`、非空率、熵和
   `token_recon_ppl`。
6. 主要效应量定义为 `shuffled_plan_ppl - oracle_plan_ppl`；正值表示正确 clean plan
   优于错误 plan。

### 判读规则

- 三个 seed 上 gap 大体稳定为正：支持 token 生成器确实使用 plan 语义，后续重点应转向
  plan prior / plan denoiser 为什么生成不了同等质量的 plan。
- gap 在共享噪声后接近零或符号不稳定：旧正结果主要可能来自采样方差，需要重新设计
  plan 注入或增加强制利用约束。
- token reconstruction 正常但 oracle/shuffled 无差异：只说明 token autoencoding 正常，
  不能说明 plan 有用。

### 修正后的结论

1. frozen clean plan 会因果性地改变 token 生成；plan 通道不是死的。
2. 正确 plan 没有稳定改善 GPT-2 generative PPL。PPL主要衡量通用流畅性，不直接衡量
   是否遵循目标 plan；错误 plan 也可能诱导出更常见、更容易被 GPT-2 预测的文本。
3. 正确 plan 对 reference 的 ROUGE-1/ROUGE-L 有一致的小幅改善，说明语义匹配信号存在，
   但当前注入强度或评价方法不足以证明强 plan adherence。
4. 旧的 PPL 正 gap 主要受未配对采样噪声影响；不能再把它当作 clean-plan 语义收益证据。
5. 下一步不应只是把同一 PPL 实验复制到更多硬件，而应加入逐样本 paired plan-adherence
   指标：生成文本与 oracle plan/reference 的 Sentence-T5 cosine、paired ROUGE delta、
   plan retrieval accuracy，以及 zero/mean/random-plan 对照。之后再做 H100、L40S、SenseCore
   的完整平衡 backend replication，不能把不同 seed 随意分散到不同硬件。

### Sentence-T5 plan-adherence 分析

使用与 plan 构造相同的 frozen `sentence-transformers/sentence-t5-xl`，对 reference、
oracle 输出和 shuffled 输出重新编码。主要量定两个效应：

- target delta：`cos(oracle output, own reference) - cos(shuffled output, own reference)`。
- 2x2 interaction：同时奖励 oracle 输出接近 own reference，以及 shuffled 输出接近
  它真正被注入的 next reference，并减去两个交叉错配分数。

| 指标 | 32 steps | 64 steps | 六格合并 |
|---|---:|---:|---:|
| target delta | `+0.005084` `[0.003750, 0.006440]` | `+0.004385` `[0.002985, 0.005775]` | `+0.004735` `[0.003739, 0.005755]` |
| 2x2 interaction | `+0.005543` `[0.004759, 0.006357]` | `+0.004489` `[0.003672, 0.005326]` | `+0.005016` `[0.004373, 0.005671]` |
| oracle own-vs-next retrieval | `57.55%` | `54.95%` | `56.25%` |
| shuffled injected-vs-own retrieval | `59.38%` | `55.34%` | `57.36%` |

六格合并的 target delta 在 `71.88%` 数据项上为正，interaction 在 `84.77%` 数据项上
为正；每个 seed 的两项均值也都为正。这说明输出确实会跟随被注入 plan 的语义身份，
但绝对 cosine 效应很小，pairwise retrieval 也只略高于50%：最准确的概括是
**plan 语义通道已激活，但 adherence 还很弱**。

这个指标与 plan 使用同一 encoder，因此对当前机制很敏感，但也存在 circular/self-metric
bias。ROUGE 的独立词法证据与它同方向；下一轮应补一个不同系列的语义 encoder
或 NLI evaluator，不应只依赖 Sentence-T5 自评。

### 为什么会“强烈改变输出，但只弱语义遵循”

1. 768维 plan 被投影成8个 plan slot，直接与 token latent 一起进入同一个12层
   bidirectional Transformer。这是很强的全局因果通路，所以换 plan 会大幅改变生成轨迹。
2. 训练中只提供正确 clean/noised plan，没有 shuffled、zero 或 hard-negative plan；也没有
   直接奖励“正确 plan 比合理但错误的 plan 更能解释当前 token”的对比/adherence loss。
3. plan MSE 要求模型从 plan slots 重建 sentence embedding，但它可以在 plan 子通路上完成，
   并不必然迫使 token field 精细区分 plan 身份。
4. token latent 本身是信息很丰富的 fixed T5-small contextual embedding，训练时仍包含
   被加噪的目标 token 信息，因此存在绕过 plan 细粒度语义的 shortcut。
5. decoder CE 行的期望比例只是 `decoder_prob=0.2`；其余0.8主要是 token denoising L2。
   这不意味中80%行完全没有 plan 梯度，但它使“直接用 clean plan 改善离散 token 预测”
   的压力相对较弱。

### 下一个最小诊断闭环

先不训新模型，在同一 checkpoint、reference 和 token RNG 上做 evaluation-only 干预：

1. oracle、shuffled、batch/global mean、zero，以及按 clean-plan norm/covariance 匹配的 random plan。
2. 做 `mean + alpha * (oracle - mean)` 及 oracle-to-shuffled interpolation，检查 adherence 是否随
   plan 剂量单调变化。
3. 主指标用 paired 2x2 semantic interaction、independent semantic score 和 paired ROUGE；PPL 只作
   fluency 副指标。

这一组能把两种解释分开：如果只要一个在 clean-plan manifold 上的向量就能带来收益，
那么当前主要学到的是 manifold/style prior；如果 oracle 和插值呈现稳定的语义剂量反应，
才能确认精细 plan identity 在控制 token 生成。只有这一诊断成立后，才值得进入
plan dropout/shuffling、contrastive adherence 或 guidance 等训练改动。

## 2026-07-15：后续架构 ablation 记录

### 暂缓：prefix/future 非对称 attention

后续需要比较 plan 与 token 的可见性拓扑，但本轮不实现，避免与参数共享 ablation 混杂。
条件生成中的目标拓扑是：plan denoiser 可以读取 noisy plan、时间和前文/prefix token，不能
读取待生成的 future token；future-token denoiser 可以读取 prefix、plan 和 noisy future。
clean plan target 的 encoder pass 仍允许读取真实 future token。还需单独检查 prefix 的
T5-small contextual embedding 是否由 `prefix only` 编码，避免双向 T5 embedding 预先泄漏后文。

计划比较三个 attention arms：

1. `joint`：当前 plan/future 双向联合去噪。
2. `hierarchical`：`plan <- prefix`，`future <- prefix + plan + future`。
3. `no_context`：plan 不读取 prefix，作为无条件 plan prior 下界。

### 当前：独立 plan denoiser

先做参数共享 ablation。`shared` 保持当前实现：plan slots 与 token field 经过同一个 ELF trunk，
再由 shared plan readout 预测 clean plan。`independent` 增加独立 plan-only DiT，只接收 noisy
plan 与 plan time；token ELF trunk 仍接收 plan slots 作为生成条件。这样生成动力学变成独立
plan prior 驱动 token denoiser，但暂不改变 token trunk 内现有双向 attention。ELF-B 的
independent arm 使用与 shared trunk 同深度、同 block 结构的12层 plan-only ELF stack，避免
把参数独立性与“更浅的小模型”混为一个变量。

核心判据：固定 plan noise/time 时，independent plan prediction 必须不随 token field 输入变化；
plan MSE 不能训练共享 token ELF blocks。实验必须从相同 source/checkpoint、seed、batch、数据、
采样设置出发，只改变 `plan_denoiser_type=shared/independent`，并同时比较 plan MSE、自由生成
gPPL/MAUVE、固定点 token L2、oracle/shuffled adherence 与生成 plan 到 clean-plan manifold 的距离。
