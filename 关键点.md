# ELF 关键点

## 两个不同的 latent

ELF sentence-plan fusion 中必须始终区分两个 latent：

1. **Token latent / token field（`x0`, `z_x`）**
   - 干净目标 `x0` 不是可学习的 token lookup embedding，也不是 plan embedding。
   - `x0` 由固定的 **T5-small encoder** 生成：取 `T5EncoderModel(...).last_hidden_state`，再按配置的 `latent_mean` / `latent_std` 标准化。
   - 因为是 T5 encoder 的 hidden state，每个 token 位置都是结合整个可见上下文后的 **contextual embedding**，而非静态词向量。
   - T5-small 在 ELF 训练中处于 `eval()` 且所有参数 `requires_grad=False`；它定义了稳定的 token latent 空间，不被 ELF 的 loss 改写。
   - 生成时 ELF 从 token noise 出发，逐步将 `z_x` 去噪到这个固定 T5 contextual-embedding 空间，再通过 decoder head 还原成 token。

2. **Plan latent（`s0`, `z_s` / `plan_z`）**
   - 句子级、全局的语义/规划 latent，不是 token field 的池化别名。
   - learned-plan 设置中，干净 plan `s0` 由 learned query 读取干净 `x0` 后产生；Sentence-T5 设置中，则由独立且冻结的 Sentence-T5 encoder 生成。
   - 推理时不使用目标文本预先编码 plan；`plan_z` 与 token latent 一样从噪声开始并逐步去噪。

## 两者的关系

```text
token latent: z_x ∈ 固定 T5-small contextual-embedding 空间
plan latent:  z_s ∈ 句子级 plan 空间

[plan slots | token-field slots]
                ↓
       共享 ELF Transformer
       ↙                 ↘
plan prediction          token prediction
```

两个 latent 的起点、目标空间、粒度和 loss 都不同，但在共享 ELF trunk 中通过双向 attention 相互影响，并在推理时联合演化。

因此，对当前 aux-pass 退化的准确表述应该是：

> 额外 plan auxiliary loss 没有改写固定 T5-small 所定义的 token latent 目标空间；它改写了共享 ELF 去噪器的参数，并可能通过坍缩的 plan latent 在联合采样中干扰 token latent 的去噪轨迹。故 token reconstruction 仍可正常，但从噪声生成高质量 token contextual embeddings 的能力可以显著下降。

## 分析时不要混淆

- `token latent` 不等于 token ID；它是一串 contextualized T5 hidden states。
- “T5-small 固定”不等于 token latent 固定不变；训练数据不同，`x0` 也不同，只是生成该空间的 encoder 参数不更新。
- `plan latent` 不是第二套 token embedding；它是更粗粒度的全局条件。
- `detach(s0)` 只是截断指定的梯度路径，不代表 plan 分支与 token 分支在参数或采样动力学上彼此隔离。

## 联合学习 latent 的必读论文

下面三篇是分析 ELF 中“latent 表示学习—diffusion prior—decoder”联合训练问题的直接参考。PDF 已下载到 `papers/latent_joint_training/`。

1. **UNITE: End-to-End Training for Unified Tokenization and Latent Denoising**
   - Duggal et al., arXiv:2603.22283
   - [arXiv](https://arxiv.org/abs/2603.22283) | [本地 PDF](papers/latent_joint_training/2603.22283_UNITE.pdf)
   - 关注点：同一 Generative Encoder 通过两种 forward pass 共享参数，同时承担 tokenization/representation inference 和 latent denoising。ELF 的 aux-pass 比例设计与共享 trunk 梯度竞争应与此对照。

2. **Unified Latents (UL): How to train your latents**
   - Heek, Hoogeboom, Mensink, Salimans, arXiv:2602.17270
   - [arXiv](https://arxiv.org/abs/2602.17270) | [本地 PDF](papers/latent_joint_training/2602.17270_Unified_Latents.pdf)
   - 关注点：latent 不只要可被 decoder 重建，还必须受 diffusion prior 约束、处于 prior 容易建模的分布中；论文通过关联 encoder output noise 与 prior minimum noise 来统一两者。

3. **How to Train Your Latent Diffusion Language Model Jointly With the Latent Space**
   - Meshchaninov et al., arXiv:2605.07933
   - [arXiv](https://arxiv.org/abs/2605.07933) | [本地 PDF](papers/latent_joint_training/2605.07933_LDLM_Joint_Latent.pdf)
   - 关注点：这是三篇中与 ELF 文本 latent diffusion 最直接的对照。论文明确报告 naive joint training 会产生低质量 diffusion model，并使用 MSE decoder loss、diffusion-to-encoder warmup、adaptive timestep sampling 和 decoder-input noise 改善训练。

### 需要固定的共同认识

> Latent 不能只按“能否重建”来评价；它还必须容易被 diffusion/flow prior 从噪声建模。反过来，diffusion loss 也不能不加控制地改写 latent encoder 或共享 trunk。联合训练的核心是平衡“表示可重建性”、“prior 可建模性”和“两类梯度的时间/强度配比”，而不是简单把多个 loss 相加。

上述“关注点”目前基于论文元数据与摘要；完整方法、loss 定义和 ablation 结论需在逐篇精读 PDF 后再形成 ELF 的实现决策。

## UNITE：共享参数、梯度与多 flow 轮数

### 共享的是什么

UNITE 不是共享整个 autoencoder，而是让同一个 **Generative Encoder** `GEθ` 承担两个角色：

```text
tokenization / reconstruction path:
    z0 = GEθ(image patches, noisy latent registers)
    x_hat = Dψ(noise(z0))

generation / flow path:
    zt = t·stopgrad(z0) + (1-t)·ε
    z0_hat = GEθ(zt, t, condition)
```

- 两条路径共享 GE 的 latent projection、time/condition modulation、Transformer attention/MLP blocks、final projection 和 output normalization。
- image patch embedding 只出现在 tokenization path；image decoder `Dψ` 是独立模块，不与 GE 共享参数。
- 两种 mode 使用相同的 latent-register 接口和带可学 affine 的 LayerNorm，论文认为 attention/MLP 是两种任务中最可复用的部分。

### 梯度路由

UNITE 的 loss 为：

```text
L = L_recon + L_flow

L_recon = L1(Dψ(noise(GEθ(x))), x) + LPIPS(...)
L_flow  = E[t,ε] ||GEθ(zt,t) - stopgrad(z0)||²
```

梯度关系：

```text
L_recon → decoder Dψ
        → GEθ 的 tokenization forward

L_flow  → GEθ 的 denoising forward
        ✕ 不经 z0 回传第一次 tokenization forward
```

`stopgrad(z0)` 的作用是阻止 `L_flow` 沿“noisy latent → clean latent → tokenization forward”的直接路径再更新一次 GE。它不会把 reconstruction 与 flow 解耦，因为两个 loss 仍在更新同一份 `θ`。

论文的 ablation 显示：

- 共享参数 + stop-gradient 是 UNITE 的主设定。
- 去掉参数共享后，encoder 和 denoiser 各自只受自己的 loss 更新，效果仍有竞争力，但 reconstruction–generation trade-off 略差。
- 在共享参数时去掉 stop-gradient，`L_flow` 既更新 denoising forward，又经 `z0` 更新 tokenization forward，论文将其解释为额外的非对称梯度干扰，中间去噪轨迹和表示对齐变差。

### “多轮”的准确含义

UNITE 不是每个训练 iteration 进行多次 reconstruction，而是：

> 每 1 次 tokenization/reconstruction forward，复用同一个 detached clean latent `z0`，独立采样 `k` 个 timestep/noise 的 flow 训练样本。

```text
1 × reconstruction batch: x → z0 → x_hat
k × independent flow samples:
    zt_i = t_i·stopgrad(z0) + (1-t_i)·ε_i
    z0_hat_i = GEθ(zt_i, t_i)
```

它不是：

- 在训练中把同一个 `zt` 连续去噪 `k` 步；
- 做 `k` 次 optimizer step；
- 进行 `k` 次 reconstruction。

作者代码的 `flow_steps_per_recon=k` 会把 `z0` 在 batch 维复制 `k` 份，用 `flow_mini_batch` 分块节省显存，最后将各块 flow loss 按样本数相加。它没有再除以 `k`，所以增加 `k` 同时也增加了 flow objective 相对 reconstruction objective 的总梯度强度。

- 公开 reproduction config 使用 `k=3`。
- 作者发布的 UNITE-B/L checkpoint 使用 `k=14`。
- 早期 ablation 比较过 `k=1/3/4/7`，增加 flow 样本数改善了 generation FID，代价是更高的训练计算量。

`flow_steps_per_recon` 不要与 diffusion 定义中的 1000 training timesteps，或推理时的 ODE NFE 混淆。

## Unified Latents：梯度策略

UL 与 UNITE 的关键差异是：UL 的 encoder `E`、latent diffusion prior `P` 和 diffusion decoder `D` **不共享参数**。

Stage 1 中：

```text
z_clean = E(x)
z_t     = α_z(t) z_clean + σ_z(t) ε
L_prior = diffusion ELBO / denoising loss on z_t

z_0     = α_z(0) z_clean + σ_z(0) ε_z
L_dec   = weighted diffusion-decoder loss D(x_t, z_0)

L = L_prior + L_dec
```

梯度路由为：

```text
L_prior → prior P
        → encoder E          # 默认不 stop-gradient

L_dec   → diffusion decoder D
        → encoder E          # 通过条件 latent z_0
```

因此 UL 中的 encoder 同时受到：

- decoder 的重建/信息保留压力；
- diffusion prior 的可建模性/码率正则化压力。

论文特意做了一个 ablation：在 prior 输入处加 stop-gradient，使 encoder 不再接收 prior gradient。这会明显损害 latent bitrate 与生成质量，说明对 UL 这种“分离模块”架构，prior gradient 进入 encoder 是关键机制，不是应默认截断的泄漏路径。

UL 的稳定性不是来自 detach，而是来自一组成套约束：

1. encoder 输出确定性 `z_clean`，然后加固定方差的 noise；不让 encoder 自由学习 variance。
2. encoder 最高精度与 prior 的 minimum noise level 绑定；主设置 `log-SNR=5`，约对应 `σ≈0.08`。
3. prior 在 Stage 1 使用 unweighted ELBO，避免 encoder 将信息藏在被 loss 折价的 noise/frequency 区域。
4. decoder loss 使用 sigmoid timestep weighting，并用约 `1.3–1.7` 的 loss factor 提高 decoder 相对 prior 的压力，防止强 diffusion decoder 忽略 latent，同时显式控制 latent bitrate。

还必须注意：UL 最强的主结果是两阶段策略。Stage 1 联合训练 `E + P + D`，学习 latent 空间；Stage 2 冻结 encoder/decoder，在固定 latent 上重新训练更大的 base diffusion prior，并改用更适合生成的 sigmoid weighting。论文的全单阶段尝试虽能稳定训练，但效果弱于该两阶段策略。

## 对 ELF 的直接启示

UNITE 的 `k` 增大后成功，不能直接推导 ELF 的 `plan_aux_passes` 也应增大，因为两者的 latent 拓扑不同：

```text
UNITE:
    同一个 z0 既必须支持 image reconstruction，
    又是 flow model 的唯一生成目标。
    reconstruction 对 z0 提供强制的信息保留/防坍缩约束。

ELF fusion:
    token latent x0 是固定 T5-small contextual embedding；
    plan latent s0 是另一个句子级 latent。
    token reconstruction 可以在忽略 plan 的情况下仍然完成，
    因而它不必然对 s0 提供 UNITE 中同等强度的防坍缩压力。
```

两个实现都会将多次 denoising/plan MSE 直接相加，所以都把“更多独立 noise/timestep 样本”和“更大的总 loss 权重”绑在一起。但 UNITE 有直接 reconstruction bottleneck、同一 latent 目标、mode-aligned 接口、output normalization 和 reconstruction noise 共同约束；ELF 的 plan 没有这些完全对等的保护。

因此 ELF 下一步应先分离两个变量：

1. 固定 `plan_loss_weight × plan_aux_passes` 的总强度，只增加独立 noise/timestep 覆盖；
2. 固定 pass 数，单独 sweep plan loss 权重；
3. 直接测量 token loss 与 plan loss 在共享 ELF blocks 上的 gradient norm/cosine；
4. 为 plan latent 增加真正的信息保留/防坍缩约束，而不是仅继续增加 MSE pass。

## Unified Latents：time sampling 与 sigmoid x-prediction 权重

UL/SiD2 的 sigmoid loss 不是在现有 `x0` MSE 后随手乘一个按线性时间定义的
window，而是先用 log-SNR 参数化噪声尺度，再把 timestep sampling、schedule Jacobian
和 loss weighting 分开。

令

```text
lambda(t) = log SNR(t)
```

在 UL/SiD2 的时间方向（`t=0` clean，`t=1` noise，因而
`d lambda / dt < 0`）下，最终作用在 `x`-prediction MSE 上的 sigmoid 权重为：

```text
L_x = E_t [
    -0.5 * d lambda(t)/dt
    * exp(b) * sigmoid(lambda(t) - b)
    * ||x - x_hat||^2
]
```

等价的 epsilon-MSE 写法使用 mirrored sigmoid：

```text
L_eps = E_t [
    -0.5 * d lambda(t)/dt
    * sigmoid(b - lambda(t))
    * ||eps - eps_hat||^2
]
```

因为

```text
||eps - eps_hat||^2 = exp(lambda(t)) * ||x - x_hat||^2.
```

所以 `x`-MSE 的 sigmoid 系数随 log-SNR 增加，但 clean 端的原始 `x` 误差同时迅速
下降；极 noisy 端又被 sigmoid 抑制。实际 loss/gradient 往往主要落在中间噪声区，
并不需要额外手工构造双 sigmoid window。

UL 还必须按阶段区分：

1. Stage 1 联合学习 encoder 与 latent prior 时，prior 保留严格 ELBO 权重，额外
   reweighting 取 `w=1`，避免 encoder 把信息藏进被折价的噪声尺度；
2. diffusion decoder 使用 sigmoid reweighted ELBO；
3. Stage 2 冻结 encoder/decoder 后重新训练 base latent prior，才使用面向生成质量的
   sigmoid weighting。

对 ELF 当前线性 flow（`t=0` noise，`t=1` clean，方向与 UL 相反）：

```text
z_t       = t * s0 + (1 - t) * noise_scale * eps
lambda(t) = 2 * log(t / ((1 - t) * noise_scale))
d lambda/dt = 2 / (t * (1 - t)) > 0
```

如果 `t` 从当前 logit-normal 密度 `q(t)` 采样，而目标仍是 schedule-invariant 的
log-SNR 积分，则 Monte Carlo estimator 还需要 importance correction：

```text
L_plan = E_{t ~ q(t)} [
    0.5 * (d lambda(t)/dt) / q(t)
    * exp(b) * sigmoid(lambda(t) - b)
    * ||s0 - s0_hat||^2
]
```

当前 `t = sigmoid(N(-1.5, 0.8^2))` 较窄，严格的 `1/q(t)` 会在尾部形成很大的
importance weight。实现前应优先考虑 bounded broad-log-SNR sampler 或按 log-SNR
分桶的 stratified sampling；若采用 clipping/normalization，必须把它标记为稳定化近似，
而不是严格 ELBO。

此外，当前 frozen-plan main pass 中 decoder rows 是 `t=1` 的离散 point mass，不能与
连续 `q(t)` 共用上述 estimator。连续时间的 plan denoising loss 应只统计 denoiser rows；
若仍需 clean-plan identity reconstruction，应作为独立 auxiliary loss 记录和加权。
